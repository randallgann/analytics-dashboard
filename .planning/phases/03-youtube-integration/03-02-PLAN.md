**1. Extend DashboardController** (`app/controllers/dashboard_controller.rb`):
- Add YouTube queries in the `index` action, alongside existing HN/Reddit queries:
  ```ruby
  @youtube_posts = SocialPost.for_platform("youtube").top_posts(5)
  ```
- Update @all_posts to include YouTube — it already uses `SocialPost.order(score: :desc).limit(10)` which will automatically include YouTube posts since they use the SocialPost model. No change needed for @all_posts.
- Add YouTube last-updated timestamp:
  ```ruby
  @youtube_last_updated = SocialPost.last_fetched_at("youtube")
  ```
- Add YouTube fetch error state read from cache:
  ```ruby
  @youtube_fetch_error = Rails.cache.read("social_fetch_error:youtube")
  ```

**2. Extend render_social_card helper** (`app/helpers/dashboard_helper.rb`):
- Update the badge rendering inside `render_social_card` to handle YouTube:
  - Change `post.hn? ? "Y".html_safe : "R".html_safe` to a three-way conditional:
    ```ruby
    if post.hn?
      "Y".html_safe
    elsif post.reddit?
      "R".html_safe
    else
      "YT".html_safe
    end
    ```
  "YT" distinguishes from HN's "Y" badge. The badge class is already dynamic (`dash-social-badge--#{post.platform}`) so `dash-social-badge--youtube` will be applied automatically.
- Update the meta_parts section to handle YouTube-specific display:
  - For YouTube posts, show "X views" instead of "X pts": Replace the existing `meta_parts << content_tag(:span, "#{post.score} pts")` line with conditional logic:
    ```ruby
    if post.youtube?
      meta_parts << content_tag(:span, "#{format_number(post.score)} views")
    else
      meta_parts << content_tag(:span, "#{post.score} pts")
    end
    ```
  - For YouTube posts, show "0 comments" only if comment_count > 0 (YouTube comment_count is always 0, so skip it):
    ```ruby
    meta_parts << content_tag(:span, "#{post.comment_count} comments") unless post.youtube?
    ```
  - The `post.author` line already renders for all platforms (author is always present). For YouTube, author contains the channel name which is correct.
  - The subreddit line already guards with `post.reddit?` so YouTube is unaffected.
  - Keep existing `time_ago_in_words(post.published_at)` line — works for all platforms.

**3. Add YouTube tab to view** (`app/views/dashboard/index.html.erb`):
- In the tab nav div (`.dash-tab-nav`), add YouTube button BEFORE the "All" button:
  ```erb
  <button data-tabs-target="btn" data-tab-id="youtube" data-action="click->tabs#select" class="dash-tab-btn">YouTube</button>
  ```
  Tab order: HN | Reddit | YouTube | All
- Add YouTube tab panel BEFORE the All tab panel. Mirror the HN/Reddit panel structure exactly:
  ```erb
  <%# YouTube Tab %>
  <div data-tabs-target="tab" data-tab-id="youtube" hidden>
    <div class="dash-tab-updated">
      <% if @youtube_last_updated %>
        Last updated <%= time_ago_in_words(@youtube_last_updated) %> ago
      <% end %>
    </div>
    <% if @youtube_fetch_error.present? %>
      <div class="dash-social-error">Unable to fetch YouTube videos</div>
    <% elsif @youtube_posts.empty? %>
      <div class="dash-social-empty">No recent mentions found on YouTube</div>
    <% else %>
      <% @youtube_posts.each do |post| %>
        <%= render_social_card(post) %>
      <% end %>
    <% end %>
  </div>
  ```
  Note: Error state check precedes empty state check (locked decision from Phase 2).
