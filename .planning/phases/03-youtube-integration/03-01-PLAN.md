**2. Create YoutubeClient** (`app/services/youtube_client.rb`):
- Mirror the HnClient pattern exactly (see `app/services/hn_client.rb`).
- Three typed error classes: `RateLimitError`, `FetchError`, `AuthError`.
- Constants: `SEARCH_URL = "https://www.googleapis.com/youtube/v3/search"`, `VIDEOS_URL = "https://www.googleapis.com/youtube/v3/videos"`, `SEARCH_TERM = "OpenClaw"`, `MAX_RESULTS = 25`, `DAYS_LOOKBACK = 30`.
- `initialize`: Load API key via `Rails.application.credentials.dig(:youtube, :api_key) || ENV["YOUTUBE_API_KEY"]`. If nil, raise `AuthError` immediately (fail fast, not mid-request). Use `ENV["YOUTUBE_API_KEY"]` (NOT `ENV.fetch`) to avoid `KeyError` — handle nil explicitly per RESEARCH.md Pitfall 6.
- `search_videos` (public method): Two-step fetch:
  - Step 1: Call `fetch_search_results` — GET to SEARCH_URL with params: `part=snippet`, `q=OpenClaw`, `type=video`, `maxResults=25`, `order=date`, `publishedAfter=30.days.ago.utc.iso8601`, `key=@api_key`. Parse response. Extract video_ids and snippet data. On 403: parse `error.errors[0].reason` — raise AuthError if "keyInvalid", raise RateLimitError if "quotaExceeded" (per RESEARCH.md Pitfall 1). On non-200: raise FetchError.
  - Step 2: Call `fetch_statistics(video_ids)` — GET to VIDEOS_URL with params: `part=statistics`, `id=video_ids.join(",")`, `key=@api_key`. Returns hash of `{ video_id => view_count }`.
  - Step 3: Merge snippet data with statistics via `normalize(snippet, view_count)`.
- `normalize` returns hash matching SocialPost columns: `{ external_id:, title:, url: "https://www.youtube.com/watch?v=#{video_id}", author: channel_name, score: view_count.to_i, comment_count: 0, subreddit: nil, published_at: }`.
- Rescue `Errno::ECONNREFUSED, Net::OpenTimeout, SocketError` as FetchError (matching HnClient pattern).
- Rescue `URI::InvalidURIError, JSON::ParserError` as FetchError.
- Require `net/http`, `uri`, `json`, `time` at top (matching HnClient).

**1. Create YoutubeSocialJob** (`app/jobs/youtube_social_job.rb`):
- Mirror HnSocialJob exactly (see `app/jobs/hn_social_job.rb`).
- `queue_as :default`
- `FETCH_ERROR_KEY = "social_fetch_error:youtube"`
- `perform` method:
  - `client = YoutubeClient.new`
  - `posts = client.search_videos`
  - Loop over posts, merge `platform: "youtube"`, `fetched_at: now`, `created_at: now`, `updated_at: now`.
  - `SocialPost.upsert(attrs, unique_by: [:platform, :external_id])` (mandatory for view_count freshness, per Phase 2 decision).
  - On success: `Rails.cache.delete(FETCH_ERROR_KEY)`, log upserted count.
- Error handling (matching Phase 2 patterns):
  - `rescue YoutubeClient::AuthError` — log, write to cache with 6-hour TTL, do NOT re-raise (missing credentials expected pre-deployment, per Phase 2 decision for Reddit).
  - `rescue YoutubeClient::FetchError` — log, write to cache with 6-hour TTL, do NOT re-raise (transient, job runs again in 6 hours).
  - `rescue YoutubeClient::RateLimitError` — log, write to cache with 6-hour TTL, do NOT re-raise (quota exceeded, wait for next run).
  - `rescue StandardError` — log, re-raise (unexpected errors trigger Solid Queue retry).
