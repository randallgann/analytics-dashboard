  <name>Task 1: Create GithubMetricJob that fetches all GitHub metrics</name>
  <files>

  <action>
    Create `app/jobs/github_metric_job.rb` following the research example closely:

    1. Class `GithubMetricJob < ApplicationJob`, queue_as `:default`.

    2. `perform` method:
       - Instantiate `GithubClient.new`
       - Set `today = Date.today`
       - Call `record_snapshot(client, today)` for point-in-time metrics
       - Call `record_commit_activity(client, today)` for time-series commit data
       - Call `record_contributor_count(client, today)` for contributor count
       - Call `record_release_info(client, today)` for latest release data
       - Top-level rescue for `GithubClient::RateLimitError` — log error, do NOT re-raise (job will run again in 6 hours)
       - Top-level rescue for `StandardError` — log and re-raise to let Solid Queue handle retry

    3. Private `record_snapshot(client, today)`:
       - Fetch `repo = client.repository`
       - `upsert(metric_type: "stars", value: repo.stargazers_count, recorded_on: today)`
       - `upsert(metric_type: "forks", value: repo.forks_count, recorded_on: today)`
       - Each of these wrapped in `safe_call` for error isolation:
         - `upsert("open_issues", safe_call { client.open_issues_count }, today)`
         - `upsert("closed_issues", safe_call { client.closed_issues_count }, today)`
         - `upsert("open_prs", safe_call { client.pull_requests_count(state: "open") }, today)`
         - `upsert("merged_prs", safe_call { client.pull_requests_count(state: "merged") }, today)`
         - `upsert("closed_prs", safe_call { client.pull_requests_count(state: "closed") }, today)`

    4. Private `record_commit_activity(client, today)`:
       - Call `stats = client.commit_activity_stats` (returns 52 weeks of daily data)
       - Return early if stats blank
       - Take last 5 weeks (`stats.last(5)`) to cover the 30-day window
       - For each week: `week_start = Time.at(week_stat[:week] || week_stat["week"]).utc.to_date`
       - Loop through `days` array (index 0=Sunday through 6=Saturday)
       - For each day: `date = week_start + day_index`, skip if `date > today` or `date < 30.days.ago.to_date`
       - Upsert with `metric_type: "commit_frequency"`, value is the day's commit count

    5. Private `record_contributor_count(client, today)`:
       - Call `stats = client.contributors_stats`
       - Return early if stats blank
       - `upsert("contributor_count", stats.count, today)` (count of unique contributors)

    6. Private `record_release_info(client, today)`:
       - Call `release = client.latest_release`
       - Return early if release nil
       - Store release date as Julian Day Number for numeric storage: `upsert("latest_release_date", release.published_at.to_date.jd, today)`

    7. Private `upsert(metric_type:, value:, recorded_on:)`:
       - Return if value nil
       - `GitHubMetric.find_or_create_by!(metric_type: metric_type, recorded_on: recorded_on) do |m| m.value = value end`
       - Rescue `ActiveRecord::RecordInvalid` — log warning, skip

    8. Private `safe_call(&block)`:
       - Yield block, rescue `GithubClient::RateLimitError` and `GithubClient::StatsUnavailableError` — log warning, return nil

    9. Write tests in `test/jobs/github_metric_job_test.rb`:
       - Stub `GithubClient.new` to return a mock client
       - Mock `client.repository` to return an OpenStruct with `stargazers_count: 100, forks_count: 50`
       - Mock issue/PR counts, commit stats, contributor stats, releases
       - Assert that `GithubMetricJob.perform_now` creates the expected GitHubMetric records
       - Test that a RateLimitError during one metric does not prevent others from saving (error isolation)
       - Test that top-level RateLimitError is caught and logged (not re-raised)
  </action>
  <verify>
    Run `bin/rails test test/jobs/github_metric_job_test.rb` — all tests pass.
    Run `bin/rails runner "GithubMetricJob.new.class"` — confirms autoloading.
  </verify>
